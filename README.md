# 管理学院毕业论文模拟答辩评委 Skill

> **💡 什么是 Skill？** Skill 是 AI 的系统提示词（System Prompt），它定义了 AI 的角色、行为和知识范围。本仓库的 SKILL.md 就是一个精心设计的提示词文件，你只需将它粘贴到 ChatGPT、Claude 等大模型里，即可让 AI 化身为专业的毕业论文答辩评委。

一个用于让 AI 扮演"毕业论文答辩评委"的 Prompt Skill，帮助学生在盲审/外审前自查论文的理论根基、研究设计、方法严谨性与学术规范。

---

## ✨ 功能特性

- 🎭 **角色扮演答辩评委** — 让 AI 扮演严谨、犀利的管理学院答辩评委，模拟真实答辩压力
- 🧩 **系统性提问框架** — 按「理论根基 → 模型设定 → 变量量表 → 研究设计 → 学术表达 → 灵魂拷问」六层递进
- ⚡ **单轮聚焦原则** — 每轮只抛 1–2 个最关键问题，给学生充分回应空间
- 🛡️ **危险信号预警** — 内置常见"外审一票否决"清单，帮助提前规避致命问题
- 📊 **分维度修改清单** — 答辩结束时自动生成「理论/方法/设计/语言/格式」五维改进建议
- 📚 **内置提问知识库** — 覆盖管理学主流研究范式（问卷实验、二手数据、扎根理论等）
- 🔄 **多平台兼容** — SKILL.md 可直接用于 ChatGPT、Claude、Kimi、DeepSeek 等主流大模型
- 🧪 **模拟对话示例** — 提供完整的虚构答辩对话参考，快速上手

## 🚀 快速开始

### 第一步：获取 SKILL.md

克隆或下载本仓库，找到 `SKILL.md` 文件。

```bash
git clone https://github.com/GZgreywolfy/management-thesis-defense-examiner.git
```

### 第二步：作为 System Prompt 喂给 AI

根据你使用的平台，操作方式略有不同：

| 平台 | 操作方式 |
|------|----------|
| **ChatGPT** | 新建对话 → 点击左下角头像 → **Custom Instructions** → 将 `SKILL.md` 全文粘贴至「What would you like ChatGPT to know about you to provide better responses?」文本框 |
| **Claude** | 新建 Project → 在 **System Prompt** 设置 → 粘贴 `SKILL.md` 全文 |
| **Kimi** | 新建对话 → 点击「+」添加自定义模板 → 粘贴 `SKILL.md` 全文作为系统提示词 |
| **DeepSeek** | 新建对话 → 在系统提示词设置中粘贴 `SKILL.md` 全文 |

### 第三步：开始模拟答辩

粘贴完成后，将你的论文摘要 / 选题描述 / 研究模型 / 答辩陈述发送给 AI，它会自动以答辩评委身份逐层提问。

**推荐的提问方式：**

> 老师好，我的选题是《……》，基于……理论构建了……模型，通过……方法收集了……数据，主要发现是……。请评委老师提问。

## 💬 完整使用示例

以下是一段虚构的模拟答辩对话节选（所有数据均为合成，不涉及真实个人或院校）：

---

### 第 1 轮：学生陈述 → 评委提问

**学生：**

> 各位老师好，我的选题是《智能推荐算法对用户内容消费深度的影响研究——基于信息觅食理论的视角》。我基于信息觅食理论（Information Foraging Theory），以内容新颖度（信息 scent 强度）为自变量，用户内容消费深度（阅读完成率 + 停留时长复合指标）为因变量，感知信息冗余为中介变量，用户认知闭合需求（Need for Cognitive Closure）为调节变量，共提出 5 个假设。通过某内容平台后端日志获取了 15,000 条用户行为数据，使用 Python 进行多水平线性回归分析。结果显示，内容新颖度显著正向影响消费深度，感知信息冗余的部分中介效应显著，认知闭合需求的调节效应在低水平显著。

**模拟评委：**

> 选题有意思，信息觅食理论在推荐算法场景下的应用确实有理论张力。我先问第一个问题——你论文里说"以内容新颖度为自变量"，这个变量的操作化定义是什么？信息觅食理论中的"信息 scent"是一个感知层面的构念，但你说用的是后端日志数据，那你如何从日志里推断用户的"感知"新颖度？这里有一个"构念-测量"匹配度的问题，请你解释一下。

---

### 第 2 轮：学生回答 → 评委追问

**学生：**

> 我通过推荐列表中用户未点击过的文章占比来衡量内容新颖度……嗯，但我没有做感知层面的操纵检验。

**模拟评委：**

> 好，这就是我要追问的点。你用一个行为层面的代理变量（未点击比例）来测量一个感知层面的构念（信息 scent），这在方法论文献中被称为"测量不匹配"（measurement mismatch）。如果你没有做任何操纵检验或预实验来验证这个代理变量的有效性，外审专家大概率会在这个问题上卡住。我的建议是：要么补充一个预实验（比如用 Likert 量表测量被试对内容新颖度的感知，然后验证其与你行为代理变量的相关性），要么将研究问题重新框定为"推荐多样性"而非"感知新颖度"，以匹配你的测量方式。你倾向于哪种方案？

## 🛠️ 辅助脚本：replace_authors.py

本仓库提供了一个 Python 辅助脚本 `replace_authors.py`，用于批量替换 Markdown 文件中的占位作者名，方便你在使用示例对话或提问库时将其中的虚构作者名替换为你自己的名字。

### 使用方法

```bash
# 查看所有可替换的占位符
python replace_authors.py --check

# 将所有占位作者名替换为指定姓名
python replace_authors.py --replace "张三"

# 仅替换指定文件中的占位符
python replace_authors.py --replace "李四" --file examples/sample-dialogue.md

# 将替换后的文件输出到指定目录（不修改原文件）
python replace_authors.py --replace "王五" --output-dir ./output
```

### 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| `--replace` | 指定要替换为的真实姓名 | 是（与 `--check` 二选一） |
| `--check` | 仅检查并列出所有占位符，不执行替换 | 是（与 `--replace` 二选一） |
| `--file` | 指定要处理的单个文件路径（默认处理所有 .md 文件） | 否 |
| `--output-dir` | 指定输出目录（默认直接修改原文件） | 否 |

> **提示：** 使用前请确保已安装 Python 3.6+。

## 🔒 隐私声明

本项目的提问逻辑提炼自真实毕业答辩场景的共性规律，但所有内容已彻底去主体化：不包含任何真实姓名、院校班级、真实选题或可识别个人的信息，所有示例对话均为虚构合成数据。原始素材从未、也不会包含在本仓库中。

**请用户注意：** 在使用本 Skill 时，不要输入包含真实个人信息（姓名、学号、导师信息等）的论文内容。如果已有论文中包含此类信息，建议先进行匿名化处理。
